## Considered But Not Flagged

- **`roll()` drops/duplicates after-context across a buffer boundary (the seam I was told to hunt hardest).** Constructed the canonical case: `before_context=0`, large `-A`, match near buf1 end, `after_context_left` carried into buf2. Fell apart: whenever after-context is pending, all complete lines up to `buf.len()` have already been emitted, so `last_line_visited` sits at the end of the last complete line and `[last_line_visited, end)` is at most a single partial tail. `context_start = preceding(buf, before_context)` counts *backward* from the end, so `context_start ≤ preceding(buf,0) ≤ last_line_visited`; thus `consumed = max(context_start, last_line_visited) = last_line_visited` and the entire pending after-context region is retained. The `before_context` vs `max_context` reduction only ever changes `consumed` when `last_line_visited < context_start` — i.e. when after-context is already exhausted — and that retained region is used solely by `before_context_by_line`, which needs exactly `before_context` lines (new code retains `before_context+1`). Correct.

- **`byte count` 262146 → 262142 (binary3/binary4).** Real observable change in `SinkFinish.byte_count`, but the binary offset (262153 / 262149) is unchanged and computed positionally, and the PR author updated the expectations with a comment ("always detect binary in the current buffer before searching"). It only affects `--stats` "bytes searched," and the smaller value is the intended, more-accurate stop-at-binary count. Intended, not a defect.

- **Extra pipe drain on `BinaryDetection::Quit`.** The inner loop now reads to fill the buffer before scanning, so on binary-quit it can drain up to ~64 KB past the binary byte from stdin/pipe before truncating `end = oldend + i` and latching quit. Over-read bytes are discarded and never re-read (next `fill()` short-circuits on the latched offset), so no correctness impact; only marginally more bytes drained from the source. The old chunked read could also over-read within a chunk. Not a defect.
